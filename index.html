This dashboard reads from data/bazaar.json and data/auctions_ended.json — static files committed to your repo by a GitHub Action that runs every 5 minutes. No CORS issues, no proxy failures.

STEP 1 — Add the workflow file

Create this file in your repo at exactly this path:
.github/workflows/fetch.yml
This tells GitHub Actions to run your fetch script every 5 minutes and commit the results.

STEP 2 — Add the fetch script

fetch_data.py
Place this at the root of your repo. It calls the Hypixel API and writes to data/bazaar.json and data/auctions_ended.json.

STEP 3 — Trigger first run manually

Go to your repo → Actions tab → "Fetch Skyblock Data" → "Run workflow" → Run. This creates the data/ folder and the JSON files for the first time. After that it runs automatically every 5 minutes.

STEP 4 — Check data freshness

data/meta.json
This file contains the last update timestamp and whether each fetch succeeded. The dashboard reads it to show data age in the header.
